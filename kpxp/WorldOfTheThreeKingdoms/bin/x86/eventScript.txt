// 雙鈄號表示註解，遊戲不會理會所有以//開頭的行

// 以#basic開始表示基本資訊的部分
#basic

// name為事件名稱，用於設置事件之間的前置事件，詳見happenedEvent
name 測試事件

// happenedEvent是前置事件，這些事件必須已發生後，這事件才能發生。填入以半型逗號分隔的事件名稱。
// 可以不填，表示沒有前置事件
happenedEvent 事件1,事件2

// maxTime表示事件最多可發生的次數。不填代表可發生無限次
maxTime 10

// scenario為有效的劇本，可填入以半型逗號分隔的劇本名稱。可以不填，則所有劇本有效
scenario 英雄集结,乱武三国

// dateStart為事件可能發生的開始日期，以年/月/日表示。可以不填，則劇本開始時即有效
dateStart 280/2/15

// dateEnd為事件可能發生的結束日期，以年/月/日表示。可以不填，則永遠有效
dateEnd 300/10/15

// chance表示發生機率，65代表每天有65%機率發生，不可以與random同時出現。
// chance及random也不填的話表示條件符合時，事件就必然發生
// chance 33

// random表示發生機率，65代表每天有65分之一的機率發生，不可以與chance同時出現。
// chance及random也不填的話表示條件符合時，事件就必然發生
random 90

// 以#condition開始表示事件觸發條件的部分，可填入個別物件的條件。
#condition

// 條件格式為
// 字母P/A/F/T序號 物件ID 條件
// 字母P、A、F、T：分別表示該物件是武將、建築、勢力及部隊
// 序號：一個任意數字，在同一事件、同一種類間需要唯一。留意字母及序號間沒有空格分隔
// 物件ID：該武將、建築、勢力或部隊的指定ID，在劇本裏可以找到。如果為-1，代表所有對應種類的物件也可觸發事件
// 條件：半型空格分隔的條件ID，可以在CommonData或劇本檔中的Condition表找到。所有條件必須符合。
// 可以留空，表示沒有條件需要符合。

// 注意：字母P/A/F/T序號 表示一個物件，在以後的部分也會用到。所有物件必須在#condition裏定義，否則無效

// 一行一個條件。所有條件必須符合，才能觸發事件

P1 -1 105 115
P2 -1 105 115
F1 -1 3043
P3 -1 802
P4 7007

// 以#conditionRelation開始表示事件觸發條件，可填入物件之間的關係條件。所有條件必須符合，才能觸發事件
#conditionRelation

// 關係條件格式為
// 字母P/A/F/T序號 關係 字母P/A/F/T序號
// 字母P、A、F、T序號：在#condition部分定義的物件及其序號。
// 關係：可為in、notin、sameas或notsameas。

// in表示前面的物件在後面的物件裏：
// 前面為武將(P)的話，後面可為建築、勢力或部隊(A,F,T)
// 前面為建築(A)或部隊(T)的話，後面只可為勢力(F)
// 前面不能是勢力(F)

// notin就是in的相反

// sameas表示兩個武將是在同一建築或部隊、或兩個建築在同一勢力。兩個物件必須為相同種類，且不能是勢力(F)或部隊(T)

// notsameas就是sameas的相反

// 一行一個條件。所有條件必須符合，才能觸發事件

P1 in F1
P2 in F1
P3 in F1
P1 sameas P2
P1 sameas P3

// 以#speech表示事件觸發時的對話
#speech

// 對話格式為
// 字母P/A/F/T序號 對話
// 字母P、A、F、T序號：在#condition部分定義的物件及其序號。
// 對話：對話內容。一行代表一個對話框。
// 如果物件為建築，說話人物會在該建築中能力最高者，如無，則由勢力君主代替。如無，則由許劭代替
// 如果物件為勢力，說話人物會為該勢力君主。如無，則由許劭代替
// 如果物件為部隊，說話人物會為該部隊主將

// 對話中如果使用「字母P、A、F、T序號」，則會以該物件的名稱取代

F1 P1及P2勞苦功高，現頒授盔甲一套
P1 感謝P3
P2 感謝P3
P4 多句對白以多行分隔
P4 即是這樣 

// 以#effect表示事件效果
#effect

// 事件效果格式為
// 字母P/A/F/T序號 效果
// 字母P、A、F、T序號：在#condition部分定義的物件及其序號。
// 效果：半型空格分隔的事件效果ID，可以在CommonData或劇本檔中的EventEffect中找到。所有效果也會加入

P1 0 10
P2 0 10
